Figure 4.2 — Matrix multiply tiling: naive vs cache-blocked

Identical MAC count. Different locality. Order-of-magnitude difference in time.

Naive — stream A once per output column
Cortex-M4, no cache. Memory-bound.
compute   163,840 MAC × 1 cyc = 163,840 cyc
memory   32,768 cyc (≈ 20% overhead)
measured ≈ 3–5 ms on 100 MHz M4
Tiled — 64×64 sub-blocks reused
Cortex-M7 + 16 KB L1. Compute-bound.
compute   163,840 MAC × 1 cyc = 163,840 cyc
memory   ≈ 500 cyc (negligible)
measured ≈ 1.7 ms on 100 MHz M7

Same matrices, same MACs. The cache-blocked pass keeps each sub-block of A in L1 while it is reused across many output elements; the naive pass streams every element from SRAM once per output column.
