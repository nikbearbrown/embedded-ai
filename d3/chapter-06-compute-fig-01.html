FLOP view — frameworks count this way

Two FLOPs per output element: one multiply followed by one add.

FLOP 1 — multiply
a × b → t
FLOP 2 — add
result + t → result

2 FLOPs · two cycles on a scalar pipeline

→
collapses to

MAC view — hardware does this

One fused multiply-accumulate instruction completes in a single cycle.

1 MAC — fused multiply-accumulate
result += a × b
one instruction · one cycle on M4 / M7 / A53

1 MAC · half the instruction count, same math

Rule of thumb. For pure multiply-add kernels (matmul, conv): MACs ≈ FLOPs ÷ 2.

Mixed work (activations, pooling) — keep FLOPs as a conservative upper bound; only fused MAD lanes save the cycle.

FLOP and MAC counts for a sample fully-connected layer.
